## Design Context

### Users
Finance and operations teams (accountants, AP clerks, procurement managers) who need to verify that Purchase Orders, Goods Receipts, and Invoices align. Used in high-volume, detail-oriented workflows where accuracy and speed matter.

### Brand Personality
Minimalist, technical, precise. The interface should feel like a professional tool — not a marketing site. Think Linear, Raycast, or a modern code editor.

### Aesthetic Direction
- **Theme**: Dark mode by default. Deep warm charcoal backgrounds, not pure black.
- **Palette**: Subtle teal accent for primary actions; functional colors (green/amber/red) for status only.
- **Typography**: Geometric sans (Space Grotesk) for structure; monospace (JetBrains Mono) for data and inputs.
- **Layout**: Flat hierarchy. No card shadows. Use 1px borders and generous whitespace to separate sections. Left-aligned, asymmetric compositions.
- **Motion**: Minimal and purposeful. Smooth opacity/transform transitions only.

### Design Principles
1. **Every pixel is intentional** — no decorative elements without function.
2. **Data first** — the interface gets out of the way so users can focus on documents and match results.
3. **Density without clutter** — tight groupings for related data, generous separations between sections.
4. **Technical honesty** — monospace for values, sharp edges, subtle grid lines, no rounded "friendly" aesthetics.
5. **Calm confidence** — no urgent reds or flashing alerts unless there's an actual error.
